1 Introduction

When performing point estimation, there isn’t just one right answer. For example, the Method of Moments (MOM) and Maximum Likelihood Estimates (MLEs) aren’t always the same.

One question that comes up, is why we calculate the sample variance as \[S_{n-1}^2 = \frac{1}{n-1} \sum_{i=1}^n (X_i -\bar{X})^2,\] when it is supposed to be the ``average squared distance from the mean, i.e., \[S_n^2 = \frac{1}{n} \sum_{i=1}^n (X_i -\bar{X})^2.\]

2 Comparing Estimators

2.1 Initial Simulation

In this lab, we will explore why we use \(S_{n-1}^2\) and not \(S_n^2\). To do this, we will generate data from four known distributions:

  • Poisson\((\lambda=2)\)
  • Binomial\(\left(n=50, p=\frac{5-\sqrt{21}}{10}\right)\)
  • Normal\((\mu=0, \sigma=\sqrt{2})\)
  • Exponential\(\left(\lambda = \frac{1}{\sqrt{2}}\right).\)

Remark: Note that I have selected the parameters so that the true variance for all four models is 2.

For each distribution, write a loop that completes the following 1000 times. Put set.seed(7272) at the top of your code chunk.

  • Generate a sample of size \(n\) from the distribution.
  • Calculate and save \(S_{n-1}^2\) and \(S_n^2.\)

Start with \(n=20\). Is there evidence that \(S_{n-1}^2\) is a better estimator than \(S_n^2\)?

Create a plot or combination of plots (using patchwork) to show the results across all four distributions. Make sure to create a corresponding table that numerically summarizes the results of the simulation. You should include \[\begin{align*} \textrm{Bias} &= \textrm{Average of Estimates} - \textrm{True Value}\\ \textrm{Precision} &= \frac{1}{\textrm{Variance of Estimates}}\\ \textrm{Mean Square Error} &= \textrm{Bias}^2 + \textrm{Variance of Estimates}. \end{align*}\]

Solution

Code 
set.seed(7272)

library(ggplot2)
library(dplyr)
library(patchwork)

#Simulation Function
simulate.variance = function(generator.fn, n, n.sim = 1000) {
  s_n1 = numeric(n.sim)
  s_n  = numeric(n.sim)
  
  for (i in 1:n.sim) {
    x = generator.fn(n)
    
    s_n1[i] = var(x)                 # S_(n-1)^2
    s_n[i]  = mean((x - mean(x))^2) # S_n^2
  }
  
  tibble(s_n1 = s_n1, s_n = s_n)
}

#Summary Function
summarize.results = function(data, dist.name) {
  true.var = 2
  
  tibble(
    Distribution = dist.name,
    Estimator = c("S_{n-1}^2", "S_n^2"),
    Bias = c(mean(data$s_n1) - true.var,
             mean(data$s_n)  - true.var),
    Variance = c(var(data$s_n1),
                 var(data$s_n)),
    Precision = c(1 / var(data$s_n1),
                  1 / var(data$s_n)),
    MSE = c(
      (mean(data$s_n1) - true.var)^2 + var(data$s_n1),
      (mean(data$s_n)  - true.var)^2 + var(data$s_n)
    )
  )
}

Solution

Code 
#Create bootstrap function
bootstrap.variance = function(x, n.boot = 1000) { 
  n = length(x)
  
  s_n1 = numeric(n.boot)
  s_n  = numeric(n.boot)
  
  for (i in 1:n.boot) { #loop sample
    x.sample = sample(x, size = n, replace = TRUE)  # bootstrap sample (replacement)
    
    s_n1[i] = var(x.sample)
    s_n[i]  = mean((x.sample - mean(x.sample))^2)
  }
  
  tibble(s_n1 = s_n1, s_n = s_n)
}

Code

pois.boot  = bootstrap.variance(x.pois)

The bootstrap distributions of S_(n-1)^2 and S_n^2 have similar shapes to those observed in the full simulation, including right-skewness and decreasing spread. This supports that bootstrapping is a useful method for approximating the sampling distribution using only one sample. However, the bootstrap distributions are centered around the sample variance rather than the population variance, so results can vary depending on the initial sample. When using different seeds, the overall shape remains similar, but the location and spread may change, reflecting variability in the original samples.

3 Executive Summary

Summarize the work you’ve done here to provide a brief (200-300 word) summary of the simulation and the results using your work in Lab 11. Your summary should be professional, clear, and all claims should be corroborated with statistical support.

Solution This lab investigated the properties of two variance estimators, S_(n-1)^2 and S_n^2, through simulation and bootstrapping across four distributions. Using the Poisson, Binomial, Normal, and Exponential models, which were all parameterized to have true variance 2, I repeatedly generated samples and computed both estimators. The results consistently showed that S_(n-1)^2 is approximately unbiased, with mean estimates very close to the true value of 2, while S_n^2 systematically underestimates the true variance. This bias was evident in the simulation summaries and density plots, where the distribution of S_n^2 was shifted slightly left of 2.

As sample size increased from n=20 to n=500, the variance and MSE of both estimators decreased, indicating improved precision. While the bias of S_n^2 diminished with larger n, it remained consistently negative. Consequently, the difference in performance between the two estimators was negligible for larger samples, though S_(n-1)^2 was unbiased for all sample sizes and did not exhibit persistent negative bias like S_n^2 did.

Density plots revealed that both estimators have right-skewed sampling distributions for small n, becoming more symmetric as n increases. Bootstrapping from a single sample produced similar distributional shapes, demonstrating that resampling can approximate the sampling distribution. However, bootstrap results were centered around the sample variance and varied across seeds, reflecting differences in the initial samples and potential inaccuracy with respect to the overall data. Overall, the simulations confirm the stated preference of using S_(n-1)^2, especially in smaller samples.
